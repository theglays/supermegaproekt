# GAME DESIGN DOCUMENT (GDD)

**Проект:** «Три карты: по следам Германна»  
**Жанр:** Point-and-Click Adventure  
**Платформа:** PC  
**Движок:** Unity 2022+ (C#)

---

## 1. КОНЦЕПЦИЯ И ЦЕЛЬ

**Цель:** Создать интерактивную цифровую карту-реконструкцию петербургских локаций повести «Пиковая дама», позволяющую пользователю исследовать художественное пространство произведения через игровые механики и элементы визуализации текста.

### Core Loop (Игровая петля):

1. **Исследование:** Игрок перемещается по локации, осматривает предметы.
2. **Взаимодействие:** Клик по объектам/NPC открывает описание или диалог.
3. **Сбор информации:** Новые факты записываются в Журнал (Дневник/Досье).
4. **Прогресс:** Сбор ключевой информации открывает переход в следующую сцену.

---

## 2. ГЕЙМПЛЕЙ И МЕХАНИКИ

### 2.1 Управление

| Действие | Ввод | Реакция системы |
|----------|------|-----------------|
| **Перемещение** | ЛКМ (Левая кнопка мыши) по полу | Персонаж идет в точку клика (автоматический обход препятствий через NavMesh). |
| **Взаимодействие** | ЛКМ по объекту/NPC | Запуск диалога, чтение описания или обновление журнала. |
| **Отмена** | ПКМ (Правая кнопка мыши) или Esc | Закрыть открытое окно (диалог/журнал) / отменить текущую цель. |
| **Журнал** | Кнопка UI «Дневник» | Открыть вкладку Дневник/Досье. |

### 2.2 Камера и Пространство

1. **Тип камеры:** Изометрическая (Orthographic).
2. **Угол:** Фиксированный ракурс (X: 30°, Y: 45°). Камера не вращается вокруг игрока.
3. **Навигация:** Игрок может ходить только по запеченной зоне (NavMesh). Стены и мебель являются препятствиями.

### 2.3 Система Дневника

**Структура:** Две вкладки.
1. **Дневник:** Хронология мыслей героя (разблокируется по ходу сюжета).
2. **Досье:** Карточки персонажей (Германн, Лиза, Графиня, Томский).

**Логика:** Записи хранятся в ScriptableObjects. При взаимодействии с объектом вызывается метод `UnlockEntry(ID)`.

### 2.4 Сценарная структура (Вертикальный срез)

#### Пролог: «Карточная ночь»

1. **Локация:** Квартира офицеров.
2. **Задача:** Узнать о безумии Германна и противоречиях в истории графини.
3. **Финал сцены:** После осмотра всех объектов и диалога открывается дверь → Кат-сцена → Загрузка Сцены 1.

## 3. ИНТЕРФЕЙС (UI/UX)

### 3.1 Общая концепция

**Стиль:** Визуализация старинной записной книжки XIX века. Все UI-элементы стилизованы под бумагу, чернила, сургучные печати.

**Цветовая палитра:**
- **Фон UI:** кремовый/слоновая кость (`#F5F0E6`)
- **Текст:** тёмно-коричневый/чернильный (`#2C2416`)
- **Акценты:** приглушённое золото (`#B8954D`) для рамок и разделителей
- **Подсветка интерактивных объектов:** тёплый янтарный (`#D4A574`)

### 3.2 Интерфейс в игре (HUD)

#### Верхняя панель (Quick Access Bar)

**Расположение:** Вверху экрана, полупрозрачная.

**Элементы (слева направо):**
1. Кнопка «Выход»
2. Кнопка «Дневник» — открывает дневник/досье

**Под верхней панелью:** Индикатор текущей цели — краткая подсказка «Осмотрите квартиру» / «Поговорите с офицерами»

#### Курсор

1. **Обычный режим:** Стандартная стрелка
2. **Наведение на интерактивный объект:** Рука + подсветка объекта
